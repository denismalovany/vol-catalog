# photos/

Тут лежать реалістичні фото деталей (а не SVG-креслення). Назви файлів
відповідають id деталі в `js/parts-data.js`:

```
photos/mavic-001-1.jpg   ← деталь mavic-001, фото #1
photos/mavic-001-2.jpg   ← деталь mavic-001, фото #2
photos/mavic-001-3.jpg
…
```

## Вимоги

- **Розмір:** мінімум 1280×900 px (4:3 або 16:9). Більше — краще.
- **Формат:** `.jpg` (переважно) або `.png`. WebP теж підходить — змініть
  розширення в `parts-data.js`.
- **Вага:** до 400 KB кожне (стиснути через `cwebp -q 80` або `jpegoptim`).
- **Контент:** деталь на нейтральному фоні (#fafbfc, #e1e8ed, або міліметровий
  папір для масштабу). Без людей, без сторонніх предметів.
- **Імена файлів:** тільки ASCII, без пробілів, з дефісом (не підкресленням).

## Як додати фото до існуючої деталі

1. Покладіть файл(и) в цю папку з іменем `<id>-<N>.jpg` (напр. `mavic-001-1.jpg`).
2. У `js/parts-data.js` додайте поле `photos: ["photos/mavic-001-1.jpg", …]`
   до відповідного запису. Перше фото показується в картці каталогу, всі —
   в галереї на сторінці деталі.
3. Якщо в деталі ще немає `photos` — фолбек на SVG-креслення, нічого не
   ламається.

## Оптимізація для GitHub Pages

`git add photos/` може помітно збільшити розмір репозиторію. Підказки:

- Тримайте роздільну здатність не більше 2400 px по довшій стороні.
- Використовуйте `jpegoptim --strip-all *.jpg` (Linux/macOS через brew).
- Для зображень з однаковим вмістом (різні кути) — обмежуйте 2–3 ракурсами,
  решту залийте окремо через Git LFS.
